## Bad Smells to Detect
Analyze the architecture for the following bad smells. For each one, provide a verdict and justification:

### 1. No API Gateway
**Definition:** The system exposes microservices directly to external clients without a centralized API Gateway to handle cross-cutting concerns (authentication, rate limiting, routing, etc.).
**Look for:** Direct client-to-service communication, absence of a gateway/proxy layer at the system boundary.

### 2. Nano Service
**Definition:** A service that is too fine-grained, performing a single trivial operation with little to no business logic, causing excessive overhead in communication and deployment.
**Look for:** Services with only one or two operations, extremely narrow responsibility, high number of services relative to system complexity.

### 3. Shared Database
**Definition:** Two or more microservices share the same database instance or schema, creating tight coupling at the data layer and violating service autonomy.
**Look for:** Multiple services pointing to the same database node, shared data stores, absence of database-per-service pattern.

### 4. Microservice Greedy (a.k.a. Mega Service)
**Definition:** A service that has taken on too many responsibilities, resembling a monolith within the microservice ecosystem. It handles multiple unrelated domains or aggregates too many operations.
**Look for:** Services with a large number of endpoints/operations, services that interact with many other services, services covering multiple business domains.

### 5. Cyclic Dependency
**Definition:** A circular chain of dependencies between two or more services (e.g., Service A calls B, B calls C, C calls A), making the system fragile and hard to evolve independently.
**Look for:** Directed dependency cycles in the service communication graph.
